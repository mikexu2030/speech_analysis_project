## 关键结论

### 1. SSL模型 (wav2vec 2.0 / HuBERT)
- 情绪精度最高 (UAR 72-74%)
- **问题**: 300M+参数，端侧不可行
- **结论**: ❌ 不适合MT9655

### 2. Transformer (AST)
- 情绪精度高 (UAR 70%)
- **问题**: 87M参数，推理慢(400ms)
- **结论**: ❌ 不适合端侧CPU

### 3. ECAPA-TDNN
- 说话人SOTA (EER 0.8%)
- 参数量适中 (6.2M)
- **问题**: 情绪弱(UAR 58%)，仅单任务
- **结论**: ⚠️ 说话人可用，情绪不足

### 4. 3D-CNN + Attention
- 情绪精度尚可 (UAR 65%)
- 参数量适中 (8.5M)
- **优点**: 频谱学习，CPU友好
- **结论**: ✅ 适合作为骨干参考

### 5. MobileNetV3
- 端侧友好 (2.5M, 20ms)
- **问题**: 情绪精度低 (UAR 52%)
- **结论**: ⚠️ 太轻量，精度不足

## 我们的方案优势

1. **单模型多任务**: 同时支持4个任务，减少端侧部署复杂度
2. **频谱学习CNN**: CPU友好，比Transformer快5-10倍
3. **CBAM注意力**: 提升特征表达能力
4. **量化友好**: INT8后仅8MB，MT9655可运行
5. **多语言**: 优先英语，支持西/法/德/意/日

## 技术路线确认

```
骨干网络: 频谱学习CNN + CBAM注意力
参数量: ~8M (FP32: 32MB, INT8: 8MB)

情绪UAR目标: 65-70%
说话人EER目标: <5%
性别Acc目标: >90%
年龄MAE目标: <10年
```

## 风险缓解

- **情绪精度不达标**: 增加SSL蒸馏或数据增强
- **模型太大**: 启用轻量版 (3M参数)
- **延迟过高**: 减层或降低输入长度
