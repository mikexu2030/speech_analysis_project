# 音频大模型文本无关情绪识别（Text-Independent SER）全面调研报告
> **调研日期**: 2026-04-29
> **调研范围**: 全球主流音频大语言模型（Audio-LLM）在 Speech Emotion Recognition（SER）任务上的能力评估
> **核心目标**: 找出支持「纯音频输入、不依赖ASR转录文本」进行情绪识别的最佳模型方案

---

- **API快速验证**: GPT-4o Audio（无需部署，按量付费）

## 二、模型详细评测

### 2.1 Kimi-Audio ⭐ 推荐

**基本信息**
- **机构**: Moonshot AI（月之暗面）
- **发布时间**: 2025年4月
- **开源状态**: ✅ 模型权重、微调代码、推理代码、Demo数据全开源
- **模型架构**: 混合架构（音频Tokenizer + 音频Encoder + 核心音频LLM + 并行文本/音频生成头 + 音频Detokenizer）
- **预训练数据**: **13M小时** 多样化开放和内部音频数据
- **模型规模**: 未公开具体参数量，属于大参数音频基础模型

**情绪识别能力**
- 在 **MELD**（多模态情绪对话数据集）上表现优秀
- 在 **VocalSound**（人声分类）上表现优秀
- 支持 **VoiceBench** 语音对话基准
- 核心优势：**连续声学向量 + 离散语义token** 的双路径表示，能同时捕获声学细节和语义信息

**接入方式**
- 开源权重可本地部署（HuggingFace）
- 支持微调代码，可在自有SER数据集上继续训练
- 自有音频评估工具包

**适用判断**: ✅ **强烈推荐**。自家生态模型，技术栈熟悉，开源可定制，预训练规模最大。

---

### 2.2 Audio-Reasoner ⭐ SER准确率最高

**基本信息**
- **机构**: 阿里（基于Qwen2-Audio）
- **发布时间**: 2025年3月
- **开源状态**: ✅ 模型权重、CoTA数据集、推理代码、Demo全开源
- **模型架构**: Qwen2-Audio（Whisper编码器 + 7B LLM）+ 结构化CoT微调
- **创新点**: 1.2M样本CoTA数据集，结构化思维链监督

**情绪识别能力（量化数据）**

| 基准 | Audio-Reasoner | Qwen2-Audio-Instruct | 提升 |
|------|----------------|---------------------|------|
| **MELD SER** | **53.9%** | 49.9% | **+8.01%** |
| MMAU-mini | 53.9 | 43.0 | +25.4% |
| AIR-Bench Chat | 优于基线 | - | +14.6% |
| CoVoST2 翻译 | 优于Gemini-1.5 | - | +30.6% |

- **MELD 53.9% 为当前开源模型SOTA**
- 在 AIR-Bench Foundation 的 Speech Emotion Recognition 子任务上得分 **60.5**

**核心优势**
- 思维链推理：模型会先分析"语调高低""语速快慢""停顿模式"等声学线索，再给出情绪判断
- 可解释性：输出包含推理过程，不只是标签
- 成本：本地部署后推理成本极低

**适用判断**: ✅ **准确率优先时首选**。推理能力强，适合需要解释情绪判断依据的场景。

---

### 2.3 Qwen2.5-Omni

**基本信息**
- **机构**: 阿里
- **发布时间**: 2025年
- **开源状态**: ✅ 开源
- **模型架构**: 统一全模态架构（文本+音频+视觉统一处理）

**情绪识别能力（量化数据）**

| 基准 | Qwen2.5-Omni | Audio-Reasoner | GPT-4o Audio |
|------|-------------|----------------|--------------|
| AIR-Bench Foundation 平均 | **67.1** | 65.2 | ~62 |
| SER 子任务 | 56.2 | 60.5 | - |
| VSC 说话人识别 | **94.5** | 88.1 | - |

- 整体音频理解最强，但纯SER略逊于Audio-Reasoner
- 优势在于**全模态统一**：同一条语音可以同时做情绪识别+ASR+说话人分离

**适用判断**: ✅ **多模态融合场景**。如果你需要同时处理音频、视频、文本，这是最佳统一架构。

---

### 2.4 GLM-4-Voice

**基本信息**
- **机构**: 智谱AI（Zhipu）
- **发布时间**: 2024年
- **开源状态**: ✅ 模型、代码开源
- **模型架构**: 语音理解 + 语音生成端到端
- **特点**: 中文优化，支持实时语音对话

**情绪识别能力**
- 支持语音理解任务，含情绪识别
- 中文场景表现优于多数英文预训练模型
- 具体SER基准数据未在公开论文中详细披露

**适用判断**: ✅ **中文情绪识别优先**。如果数据集以中文为主，GLM-4-Voice的预训练语言优势可能带来更好效果。

---

### 2.5 OpenS2S / BLSP-Emo

**基本信息**
- **机构**: 中科院自动化研究所（CASIA）
- **发布时间**: 2025年7月
- **开源状态**: ✅ 全开源（模型、数据、代码、权重）
- **模型架构**: BLSP-Emo（语音理解）+ 流式交错解码（语音生成）

**独特价值**
- **专为共情语音交互设计**：不只是识别情绪，还能生成带有情感的语音回复
- 三阶段训练：语音理解预训练 → 语音生成预训练 → 共情语音指令微调
- 自动数据构建Pipeline：低成本合成多样化高质量共情语音对话
- 50k英文 + 50k中文 共情语音样本

**情绪识别能力**
- 基于BLSP-Emo的语义对齐 + 情绪对齐预训练
- 能同时理解语义内容和副语言情绪线索

**适用判断**: ✅ **需要情绪回复的交互场景**。如果你的系统不只是识别情绪，还要做出情感化回应（如客服、陪伴机器人），这是最佳方案。

---

### 2.6 GPT-4o Audio（闭源API）

**基本信息**
- **机构**: OpenAI
- **发布状态**: 闭源，API按量付费
- **架构**: 端到端多模态（文本+音频+图像统一神经网络）
- **特点**: 原生音频理解，非"ASR+文本LLM"的级联方案

**情绪识别能力**
- 能直接观察**语气、语调、背景噪音、多人对话**
- 支持笑声、歌唱、情感表达等输出
- 在多个多模态情绪理解基准上表现优异
- 无公开MELD/IEMOCAP等标准SER基准的精确数值

**成本**
- 音频输入: ~$0.006/分钟
- 文本输出: 标准GPT-4o定价

**适用判断**: ⚠️ **快速原型验证**。无需部署、无需维护，适合MVP阶段验证想法。规模化后成本较高，且不可控。

### 2.7 MERaLiON-AudioLLM

**基本信息**
- **机构**: 新加坡A*STAR I2R
- **发布时间**: 2024-2025年
- **开源状态**: ✅ 模型、权重、vLLM插件全开源
- **架构**: Whisper-large-v2编码器 + MLP适配器 + SEA-LION V3文本解码器
- **训练数据**: 26万小时音频，6200万条多模态指令

**情绪识别能力**
- MELD-Sentiment 测试得分 **42.3**（低于Qwen2-Audio的53.5）
- 强项：**新加坡英语、Singlish、多语言口音**
- 专用情绪与性别分析模块（Emotion & Gender Analysis）

**性能指标**
- 单H100 GPU: TTFT 0.149s, ITL 16ms, 867 tokens/s
- 支持16并发请求

**适用判断**: ⚠️ **东南亚/多语言口音场景**。通用SER不是最强，但区域化适配优秀。

## 六、文本无关情绪识别的核心挑战

### 6.1 为什么"文本无关"很重要？

传统SER流水线：
```
音频 → [ASR] → 文本 → [文本LLM] → 情绪标签
```

**问题**：
1. **信息丢失**: ASR丢弃了韵律、音色、语速等关键情绪线索
2. **ASR误差**: 情绪状态下语音往往更难识别（愤怒时语速快、悲伤时含糊）
3. **语言依赖**: ASR质量随语言变化，小语种基本不可用
4. **非语义声音**: 叹息、笑声、啜泣等非语言声音被完全忽略

**文本无关方案**：
```
音频 → [Audio-LLM] → 情绪标签
```

优势：直接建模声学-情绪映射，保留所有副语言信息。

### 6.2 当前模型的局限

| 局限 | 说明 |
|------|------|
| 训练数据偏斜 | 大多数预训练数据侧重ASR和语义理解，副语言（情绪）数据占比低 |
| 情绪粒度粗 | 多数模型输出离散类别标签，不支持情绪强度、混合情绪 |
| 上下文短 | 单次输入通常<30秒，长对话情绪跟踪能力弱 |
| 文化差异 | 预训练以英文为主，中文情绪表达的文化细微差别捕捉不足 |
| 可解释性 | 除Audio-Reasoner外，多数模型是黑盒输出 |

## 七、推荐实施方案

### 7.1 方案A：Kimi-Audio 主方案（推荐）

**理由**:
- Moonshot自家生态，API/SDK熟悉度高
- 预训练规模最大（13M小时）
- 开源可本地部署，可控性高
- 连续+离散双路径表示，声学细节保留充分

**实施路径**:
1. 通过Kimi API或本地部署调用音频理解能力
2. 设计专用Prompt："分析这段音频中说话者的情绪状态，只基于声音特征（语调、语速、音量），忽略文本内容"
3. 在MELD/IEMOCAP上微调（如需要更高精度）

### 7.2 方案B：Audio-Reasoner 精度优先

**理由**:
- MELD SOTA（53.9%）
- 思维链推理提供可解释性
- HuggingFace生态成熟，部署文档完善

**实施路径**:
1. HuggingFace下载 `Audio-Reasoner` 权重
2. 使用vLLM/Transformers进行高效推理
3. 利用CoT输出获取情绪判断依据

### 7.3 方案C：多模型集成

**架构**:
```
音频输入
    ├──→ Kimi-Audio → 情绪预测A + 置信度
    ├──→ Audio-Reasoner → 情绪预测B + 推理过程
    └──→ [投票/加权融合] → 最终情绪标签
```

优势：兼顾准确率、可解释性、鲁棒性。

## 八、Prompt策略（文本无关情绪识别专用）

### 8.1 基础Prompt模板

```
你是一位专业的语音情绪分析专家。

请分析以下音频片段中说话者的情绪状态。
重要：你的判断必须仅基于声音特征（音调高低、语速快慢、音量变化、停顿模式、音色质地），不要依赖语音的文本内容。

请回答：
1. 主导情绪：[Neutral/Joy/Sadness/Anger/Fear/Disgust/Surprise]
2. 情绪强度：1-5分
3. 关键声学线索：[列出你观察到的声音特征]
4. 置信度：高/中/低

音频：{audio_file}
```

### 8.2 思维链Prompt（Audio-Reasoner风格）

```
请逐步分析这段音频的情绪：

步骤1 - 声学特征提取：
- 基频（F0）轮廓：上升/下降/平稳？
- 语速：快/正常/慢？每分钟约多少字？
- 能量包络：大声/柔和/波动？
- 频谱特征：明亮/暗淡/尖锐/浑厚？

步骤2 - 情绪推断：
- 基于以上声学特征，哪种情绪最匹配？
- 是否有混合情绪迹象？

步骤3 - 输出结论：
- 情绪标签：
- 置信度：
- 推理摘要：
```
