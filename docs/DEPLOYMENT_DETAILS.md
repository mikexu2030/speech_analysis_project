# 部署详细文档

## 端侧部署流程

```
PyTorch FP32 Model (363MB)
    ↓
┌─────────────┐
│  ONNX Export │ → 1.5MB (仅任务头)
└─────────────┘
    ↓
┌─────────────┐
│  INT8量化   │ → 361MB (backbone权重)
└─────────────┘
    ↓
┌─────────────┐
│ TFLite Export│ → 端侧格式
└─────────────┘
    ↓
MT9655 TV SoC 推理
```

## 性能基准

### PC端 (CPU)
| 模型 | 推理时间 | 吞吐量 |
|------|---------|--------|
| FP32 ONNX | 8.22ms | 121 samples/sec |
| INT8 ONNX | 36.85ms | 27 samples/sec |

### 预估TV端 (MT9655)
- **总推理时间**: 200-400ms (含预处理)
- **内存占用**: ~400MB
- **功耗**: 适中

## 集成方式

### 1. 云端API (快速验证)
- **GPT-4o Audio**: $0.006/分钟
- **Audio-Reasoner**: 本地部署, 推理成本极低

### 2. 端侧SDK (产品化)
- **TFLite Runtime**: 轻量, 无依赖
- **ONNX Runtime**: 跨平台, 性能优
- **自研推理引擎**: 极致优化

### 3. 混合架构 (推荐)
```
本地端: 轻量模型实时检测 → 触发阈值
    ↓
云端: Audio-LLM精准确认 → 详细分析
    ↓
本地端: 结果缓存 → 快速响应
```
